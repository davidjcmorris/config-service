# Context Framework

This project uses a context framework stored in the `memory/` folder.
Read these files at the start of every new conversation to understand the project before doing any work.

## Files

- `@memory/ABOUT.md` — project purpose, domain model, personas, and scope
- `@memory/ARCHITECTURE.md` — architectural patterns and key decisions for both the backend and frontend
- `@memory/TECHNICAL.md` — language versions, dependencies, dev commands, database schema, and API reference
- `@memory/TESTING.md` — testing stack, conventions, patterns, and anti-patterns for both components

## Coding Standards
See `@.clinerules/coding.md` for coding standards and anti-patterns to avoid.
